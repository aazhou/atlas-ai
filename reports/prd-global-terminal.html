1产品定位

一句话：Atlas AI 是龙哥团队内部使用的全球金融AI投研终端——覆盖A股/港股/美股/加密四大市场，以DuckDB为数据底座、cron自动采集、纯静态前端部署，一屏纵览全球资产。

目标用户：龙哥 + Atlas 团队内部（不对外，无注册/登录需求）

核心价值：

  • 一个页面看完全球市场状态（跨市场仪表盘）
  • 每个市场的核心子页面完整保留并整合
  • 数据全自动（cron采集 → DuckDB → JSON → 前端直接fetch）
  • 暗色专业终端风格，信息密度高，一屏决策

5数据流设计

5.1 整体架构

cron脚本(Python) ──→ DuckDB 写入 ──→ JSON导出 ──→ git push ──→ GitHub Pages / Aliyun
  （每5/10分钟）     （本地）        （data/目录）    （自动）       （静态CDN）

铁律：数据底座是 DuckDB，JSON 只是给静态网站的导出层。所有采集脚本写 DuckDB，不直接写 JSON。JSON 由脚本在 DuckDB 写入后导出。网站只读 JSON，不直接访问 DuckDB（GitHub Pages 无法运行 Python）。

5.2 数据表设计（新增）

表名DB字段说明
hk_indicesatlas.duckdbdate, time, symbol(HSI/HSCEI), price, chg港股指数快照
hk_watchatlas.duckdbdate, time, code, name, price, chg, volume港股自选股（已有hk_watch_kline.py）
us_indicesatlas.duckdbdate, time, symbol(NDX/SPX/DJI/VIX), price, chg美股指数快照
us_watchatlas.duckdbdate, time, ticker, name, price, chg, volume美股自选股
cross_marketatlas.duckdbdate, time, market(cn/hk/us/crypto), idx_name, price, chg, chg_pct跨市场汇总（供首页仪表盘）

5.4 前端数据流

// 首页跨市场仪表盘
fetch('data/cross-market.json')
  → { markets: [{name, idx, price, chg_pct, signal}, ...] }
  → 渲染4市场卡片 + 热力图

// 各市场子页面（以港股为例）
fetch('../data/hk/indices.json')
fetch('../data/hk/watch.json')
  → 渲染恒指走势 + 自选股列表
路径注意：根目录页面用 data/...，子目录页面（hk/, us/）用 ../data/...。GitHub Pages 项目站点路径为 /atlas-ai/...，不能用绝对路径。

7UI/UX 规范

7.1 设计原则

  • 暗色终端风：深色底 + 高对比文字 + 黄/金主色调（#f0b90b）= 金融终端专业感
  • 信息密度：一屏看完全球市场核心指标，不翻页
  • 数据即时：每5分钟自动刷新，时间戳可见
  • 绿涨红跌：全站统一 PnL 颜色（#22c55e / #ef4444）
  • 不花哨：少动画、少渐变、少装饰。实用第一

7.2 色彩变量

--canvas:      #0a0a0a    /* 页面底色 */
--surface:     #121212    /* 卡片/面板底色 */
--surface-2:   #1a1a1a    /* 次级面板 */
--hairline:    #2a2a2a    /* 边框/分隔线 */
--primary:     #f0b90b    /* 主色调：黄金/亮黄 */
--ink:         #e8e8e8    /* 正文 */
--muted:       #999       /* 次要文字 */
--accent-green:#22c55e    /* 涨/正 */
--accent-red:  #ef4444    /* 跌/负 */
--accent-blue: #3b82f6    /* 链接/信息 */

市场标识色用途

🇭🇰 港股#eab308（金）卡片边框、图表线

7.4 字体

  • 正文/标题：Inter + Noto Sans SC（中文回退）
  • 数字/价格：JetBrains Mono（等宽，方便对比数字）

7.5 响应式断点

  • 桌面（>640px）：多列布局，max-width 1320px
  • 平板（480–640px）：双列布局，内边距调整
  • 手机（≤480px）：单列布局，导航简化（只显示SVG图标）

7.6 SVG图标规范

铁律：导航所有图标使用 inline SVG，禁止 emoji（跨平台渲染不一致）。SVG使用 stroke="currentColor" 跟随主题色。14x14尺寸，viewBox="0 0 24 24"，stroke-width=2。
